# 微信云托管部署指南

> 把后端从 Render（美国服务器）迁移到微信云托管（国内服务器），解决抖音 IP 限制问题。

---

## 前置条件

- **需要一个已注册的微信小程序**（不能用测试号 AppID）
  - 如果还没有：去 [微信公众平台](https://mp.weixin.qq.com) → 立即注册 → 小程序 → 个人主体（免费）
  - 注册完成后拿到 **AppID**
- **代码已推送到 GitHub**（之前已经推过了）

---

## 第一部分：开通微信云托管

### 1. 登录云托管控制台

1. 打开 **[微信云托管控制台](https://cloud.weixin.qq.com/cloudrun)**
2. 用微信扫码登录
3. 登录后会提示选择「小程序/公众号」→ 选择你注册的小程序

### 2. 创建环境

第一次进入会提示新建环境：

1. 填环境名称，比如 `video-extractor`
2. 系统会自动生成一个环境 ID（类似 `video-extractor-xxxxx`）
3. **记下这个环境 ID**，后面小程序要用
4. 资源选择：选最低配即可（0.25核 0.5G 起步，后面可以调）
5. 点确认创建

> 环境是基于腾讯云的，会复用你小程序的主体信息创建腾讯云账号。

### 3. 创建服务

1. 进入控制台主页面，点击右上角 **「新建服务」**
2. 填写：
   - 服务名称：`video-extractor`
   - **开启「允许公网访问」** ✅
3. 点确认创建

---

## 第二部分：部署后端代码

### 方式 A：GitHub 自动部署（推荐）

1. 在服务详情页，点击 **「版本管理」** 或 **「部署」**
2. 选择 **「GitHub」** 作为代码来源
3. 授权 GitHub 访问权限
4. 选择你的仓库 `wechat-video-extractor`
5. 分支选 `main`
6. 构建配置：
   - **Dockerfile 路径**：`backend/Dockerfile`
   - **构建上下文**：`backend`
7. 点 **「部署」**
8. 等待构建完成（约 5-10 分钟，需要装 Python + ffmpeg + Playwright）

### 方式 B：手动上传代码

1. 先把 `backend/` 目录打包成 zip
2. 在服务详情页点 **「上传代码」**
3. 选择 zip 文件上传
4. 等待构建

### 验证部署成功

构建完成后，服务详情页会显示一个 **公网域名**，类似：
```
https://video-extractor-xxxxx.tcloudbaseapp.com
```

用浏览器访问 `https://你的域名/api/health`，返回 `{"status":"ok"}` 就说明部署成功了。

## 第三部分：配置资源规格（重要！）

> Playwright + Chromium 需要较多内存，默认最低配可能不够。

1. 进入服务详情 → **「服务设置」**
2. 把规格调整为：
   - **CPU**：至少 1 核
   - **内存**：至少 2 GB
3. 保存

> 费用参考：1核2G 全天运行约 85 元/月。但如果设置「缩容到 0」（无请求时自动停止），实际费用会低很多，个人用大概 10-30 元/月。

## 第四部分：配置小程序

### 1. 修改 app.js

打开 `miniprogram/app.js`，把 `baseUrl` 改成你的云托管公网域名：

```javascript
App({
  globalData: {
    // ★★★ 换成你的云托管公网域名 ★★★
    baseUrl: 'https://video-extractor-xxxxx.tcloudbaseapp.com',
    chunkSize: 2 * 1024 * 1024,
  },

  onLaunch() {
    console.log('视频提取音频小程序启动');
  },
});
```

### 2. 微信开发者工具配置

1. 打开微信开发者工具
2. 导入项目，路径选 `video_to_audio/miniprogram/`
3. 填你的 **真实 AppID**（不是测试号）
4. **详情 → 本地设置 → 勾选「不校验合法域名」**
5. 按 **Ctrl + B** 重新编译

### 3. 配置 wx.cloud.callContainer（✅ 免域名、免备案）

> **方案已升级**：小程序通过 `wx.cloud.callContainer` 经微信内网调用云托管服务，**不需要公网域名、不需要备案**。
>
> 之前使用公网测试域名 `xxx.sh.run.tcloudbase.com` 会遇到腾讯云「风险提醒」中间页，导致小程序报 `连接断开 / 服务器忙 / cmdId 1006, errCode -1`。改用 callContainer 后彻底解决。

#### 步骤

1. 登录 [微信云托管控制台](https://cloud.weixin.qq.com/cloudrun)
2. 左上角找到**环境 ID**（类似 `prod-xxxxx` 或 `video-extractor-xxxxx`），复制下来
3. 打开 `miniprogram/app.js`，把 `cloudEnv` 替换成你的环境 ID：
   ```js
   globalData: {
     cloudEnv: 'prod-xxxxxxxx',           // ← 改成你的云托管环境 ID
     containerService: 'video-extractor', // 云托管服务名（创建服务时填的名字）
   }
   ```
4. 确认服务名 `containerService` 和你在云托管控制台创建的服务名一致（默认是 `video-extractor`）

#### 原理对比

| | 旧方案（公网域名） | 新方案（callContainer） |
|---|---|---|
| 路径 | 小程序 → 公网域名（被中间页拦截）→ 后端 | 小程序 → 微信内网专线 → 后端 |
| 需要域名 | ✅ 需要备案的自定义域名 | ❌ 不需要 |
| 服务器域名白名单 | 需要配 request/uploadFile/downloadFile | 不需要 |
| 费用 | 域名费 + 备案时间 | 免费、立即生效 |
| 安全性 | 暴露在公网 | 微信内网，自带防护 |

## 常见问题

### Q: 云托管和云开发有什么区别？
**云托管**：部署 Docker 容器，支持任意语言，适合已有后端项目。
**云开发**：Serverless 函数 + 云数据库 + 云存储，适合从零开发。
本项目已有 Python 后端，所以用云托管。

### Q: 需要域名备案吗？
**不需要！** 项目已改用 `wx.cloud.callContainer` 通过微信内网调用后端，不需要公网域名、不需要备案。

### Q: 真机报错「连接断开 / 服务器忙 / cmdId 1006, errCode -1」怎么办？
这通常是因为 `wx.cloud.init()` 的环境 ID 没配对，或服务名不匹配。
1. 检查 `miniprogram/app.js` 里 `cloudEnv` 是否和云托管控制台的环境 ID 一致
2. 检查 `containerService` 是否和云托管控制台的服务名一致
3. 确认后端代码已部署最新版本（接口已改为 JSON body）

### Q: Playwright 在云托管上跑不了怎么办？
如果资源不够，可以把资源规格调到 2核 4G。或者去掉 Playwright，改用其他方式获取 Cookie。

### Q: 部署后抖音还是报 Cookie 错误？
查看云托管日志，看 Playwright 是否正常获取到 Cookie。如果 Playwright 失败，可能是资源不足导致 Chromium 启动失败。
